# SteamLib Tracker

Утилита для автоматизированного сбора статистики аккаунтов Steam и экспорта аналитических срезов в Google Таблицы. Скрипт работает в фоновом режиме, сохраняет исторические отсечки по игровому времени в локальную базу данных SQLite и выгружает актуальное состояние в облако.

## Подготовка окружения

Для работы проекта потребуется Python 3 и несколько внешних зависимостей. Установи их через pip:

```bash
pip install requests pandas pygsheets python-dotenv schedule

```

Тебе понадобятся два ключа доступа. Первый — это Steam Web API Key, который можно получить на портале разработчиков Steam. Второй — сервисный аккаунт Google Cloud для взаимодействия с Google Таблицами. Скачанный JSON-ключ сервисного аккаунта нужно переименовать в `service_account.json` и положить в корневую папку проекта.

**Важно:** Обязательно открой свою Google Таблицу в браузере, нажми "Поделиться" и выдай права редактора email-адресу твоего сервисного аккаунта (он указан внутри json-файла). Иначе скрипт не сможет читать и записывать данные.

Создай в корне проекта файл `.env` и пропиши в нем доступы:

```text
STEAM_API_KEY=твой_ключ_от_steam
TABLE_NAME=Название_Твоей_Таблицы_В_Google_Drive

```

## Структура Google Таблицы

Таблица выступает в роли панели управления (админки) и конечного дашборда. Перед первым запуском скрипта внутри документа должны быть созданы два обязательных листа.

Лист **Config** отвечает за маршрутизацию аккаунтов. В ячейке A1 напиши заголовок `Club`, а в B1 — `Nickname`. Со второй строки заполняй данные: к какому клубу (зоне) относится аккаунт и какой у него кастомный ник или SteamID. Если аккаунта нет в этой таблице, скрипт не будет собирать по нему статистику.

Лист **Игры** служит фильтром для итоговой выгрузки. В ячейке A1 укажи заголовок (например, `Название`), а ниже впиши точные названия игр, которые ты хочешь видеть в отчетах. Названия должны совпадать с тем, как они написаны в Steam (с учетом регистра и спецсимволов). Все остальные игры, найденные на аккаунтах, осядут в локальной базе данных, но не будут засорять листы клубов.

## Запуск и архитектура

Запуск осуществляется вызовом основного файла:

```bash
python main.py

```

При старте скрипт читает конфигурацию из таблицы, проверяет новые ники через API Steam, резолвит их в постоянные SteamID и кэширует в локальной `steam_stats.db`. Затем происходит опрос библиотек каждого активного аккаунта. Собранные данные (минуты в игре) записываются как исторические отсечки.

На финальном этапе скрипт автоматически создает или очищает листы с названиями клубов в твоей Google Таблице и выгружает туда самую свежую статистику, отфильтрованную по заданному пулу игр. После этого программа уходит в режим ожидания и повторяет цикл каждые 4 часа.
